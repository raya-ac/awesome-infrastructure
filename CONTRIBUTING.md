# Contributing to Awesome Infrastructure

Thank you for your interest in contributing! Please ensure your pull request follows these guidelines.

## Quality Standard

To be added to this list, the project should:

- Be actively maintained
- Have a stable release or be widely used in production
- Have documentation in English
- Have a clear license

## Adding an Entry

1. **Search previous suggestions** to avoid duplicates
2. **Add the entry to the appropriate section** in alphabetical order within that section
3. **Use the following format**: `[name](link) - Description.`
   - Description must end with a period
   - Keep descriptions concise (not longer than necessary)
4. **Check your spelling and grammar**
5. **Create a pull request** with a clear title

## Pull Request Process

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes following the format above
4. Submit a pull request

## Updating Your Pull Request

If your PR is marked as needing changes, please:

1. Make the requested changes
2. Commit them to your branch
3. The PR will update automatically

## Code of Conduct

- Be respectful and constructive
- Focus on the technical merits of suggestions
- Welcome newcomers and help them learn
